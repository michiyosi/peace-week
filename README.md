## v8.0 本番データ保存（Supabase接続）

- **投稿はSupabaseに保存され、全員に共有されます**（コンセプトデモ卒業）。起動時に全ピースを取得、12秒ごとのポーリングで他の参加者のピースがリアルタイムに増えていく
- ダミーピースは撤去（0からスタート）。`?demo=1` を付けると従来のオフラインデモ（ダミー132・保存なし）
- 同じ場所の取り合いはサーバ側で検出（idx一意制約）→ 自動で近くの空きに置き直し
- **管理の削除はサーバ連動**: 削除モードでピースをタップ→サーバ上で非表示化（`hide_piece` RPC・管理パスワードで保護）。データはSupabaseダッシュボードの Table Editor でも直接閲覧・編集可能
- スキーマ: `pieces` テーブル＋RLS（誰でも投稿・閲覧可 / 更新削除は不可）＋ `admin_secret` ＋ `hide_piece(p_id, p_pass)`。文字数・色形式・idx範囲はDB側でも検証
- 運用メモ: 無料プランは1週間無アクセスで休眠するため、閑散期はGitHub Actionsの定期ping等で維持（8月は不要）

## v7.0 管理モード＆スマホ最適化（現行）

- **管理モード（v7.1〜 パスワード式）**: **ロゴを5回連続タップ**、または `?admin=1` を付けて開くとパスワード入力画面が出ます。正解するとそのタブの間は管理モード（「× 終了」で解除）。パスワードは運営内で共有し、ソースには**SHA-256ハッシュのみ**保持。変更手順: ブラウザのコンソールで `PW.hash('新パスワード')` → 出力された値を index.html 内の `ADMIN_HASH` に貼り替え。※クライアント側の簡易ロックのため厳密な防御ではありません（本番はサーバ側認証に置換）。ADMINバーから: **📊統計ダッシュボード**（総数/達成率/日別/テーマ別/色分布）・**🎨プロンプト抽出**・**🗑削除モード**（ピースをタップ→確認→削除＝モデレーション）・**⬇CSV/JSONエクスポート**・**✨お披露目プレビュー**・**↺リセット**
- **一般画面のスリム化**: 運営ボタンを管理側へ移設し、公開UIは「＋私も未来を描く／つかいかた／リンクを共有」のみ
- **スマホ最適化**: 560px以下で余白・盤面マージン・ボタン幅（CTA全幅）・ポップオーバー幅・ウィザードカード・ピッカーのタップ領域を調整
- チュートリアル抑制は `?tut=0`（管理と併用: `?admin=1&tut=0`）

## コンセプト (v3.0〜 円形構造)

一人ひとりの「好きな色」と「2045年へのメッセージ」がピースになり、**中心の光（2045）から円になって外へ広がっていく**。810ピースがそろったら、みんなの回答をもとにした一枚の絵「2045年の広島」が **8/6のお披露目会で初公開**（このデモは集まっていく過程の可視化。絵の生成は回答が出揃ってから1回のみ、という運用方針）。キーワード:「広がる」「カラフル」「光」「パズルのピース」。

## つかいかた

色とメッセージをえらんで「ピースを置く」→ ピースが円の内側から順にはまる。ピースをタップするとその人のメッセージが読める。「8月6日、いっきにお披露目」で当日の演出（渦状に完成→光のリング）をプレビュー。

## 技術メモ

- 単一HTML。外部依存は Google Fonts のみ（Zen Maru Gothic / Yusei Magic）。Canvas2D 描画。
- 円形ジグソー: **15リング × (12,18,24,…,96)セクター = ちょうど810ピース**。リング内の隣接ピースはラジアル辺の「きのこ型」ノブ（対称プロファイル）で必ず噛み合う。リング境界は円弧。各リングはランダム位相で自然な見た目に。
- 埋まり順: 内側リングから外へ（リング内はランダム）= 「丸からどんどん広がる」を保証。
- 光の演出: 埋まった半径に追従する暖色グロー、中心コア（2045）、周囲を漂う装飾ピース、お披露目時の光リング＋紙吹雪。
- URLパラメータ: `?rm=1`（reduced-motion強制）/ `?seed=N`（初期ピース数）
- テストフック: `PW.place(色,msg,名)` / `PW.assemble()` / `PW.reset()` / `PW.fill(n)` / `PW.count()` / `PW.target` / `PW.version`
- 対応: モバイル / prefers-reduced-motion / IME変換確定 / Esc・focus-visible

## 更新手順

`index.html` を編集して main にコミットすると、GitHub Pages が自動再ビルド（数分）。反映確認は `?v=日付` を付けると確実。デプロイが失敗した場合は Actions から Re-run（または空コミット）。
