## v8.7 荒らし対策・規約整備（現行）

**⚠ 公開前に必須の作業が2つあります。**

### 追加した内容

- **投稿レート制限（P0-1）**: 同一IPから10分3件を上限とするPostgresトリガー。超過はHTTP 429 → サイト側で自動巻き戻し＋「短い時間に何度も投稿はできません」を表示。閾値はSQL内の定数2つで変更可。会場Wi-Fi対策として許可リストも用意（既定は空）
- **IPの隔離**: IPは `pieces` に入れず専用の `rate_log` テーブルへ。RLSを有効にしてポリシーを1つも作らないことで、anonからは物理的に到達不能。`Prefer: return=representation` 経由で漏れる経路も作らない
- **緊急一括削除（P0-1c）**: ADMINバーに「🧹 一括削除」を追加。直近10分〜24時間を選ぶと消える枚数をプレビューし、まとめて非表示化。サーバは `hide_pieces_bulk` RPC（既存の `hide_piece` を内部で呼ぶのでパスワード検証の実装に非依存／全件削除は安全装置で拒否）。**810枚埋められた時に810回タップする事態を防ぐための復旧手段**
- **調査用ビュー**: `rate_log_summary` でIP別の投稿数を一望（SQL Editorから。anonからは不可視）
- **規約（P0-3の一部）**: IPアドレスの収集・利用目的・保持期間（2026年9月末目安で削除）を明記 ※レート制限の実装により収集項目が変わったため、旧文面は事実と食い違っていた。未成年の同意を「了解」→「保護者の同意を得たうえで」に強化し、同意チェック欄にも明記。連絡先は目立つプレースホルダ＋警告に
- **文言の保険（P1-4）**: 「810ピースそろったら8月6日にお披露目」という**達成を条件にした公開の約束**を全廃。「8月6日、集まった想いがひとつの絵に」へ。空白は「これから先の誰かの場所」として2045年の物語に接続。当日何枚でも破綻しない

## つかいかた

色とメッセージをえらんで「ピースを置く」→ ピースが円の内側から順にはまる。ピースをタップするとその人のメッセージが読める。「8月6日、いっきにお披露目」で当日の演出（渦状に完成→光のリング）をプレビュー。

## 技術メモ

- 単一HTML。外部依存は Google Fonts のみ（Zen Maru Gothic / Yusei Magic）。Canvas2D 描画。
- 円形ジグソー: **15リング × (12,18,24,…,96)セクター = ちょうど810ピース**。リング内の隣接ピースはラジアル辺の「きのこ型」ノブ（対称プロファイル）で必ず噛み合う。リング境界は円弧。各リングはランダム位相で自然な見た目に。
- 埋まり順: 内側リングから外へ（リング内はランダム）= 「丸からどんどん広がる」を保証。
- 光の演出: 埋まった半径に追従する暖色グロー、中心コア（2045）、周囲を漂う装飾ピース、お披露目時の光リング＋紙吹雪。
- URLパラメータ: `?rm=1`（reduced-motion強制）/ `?seed=N`（初期ピース数）
- テストフック: `PW.place(色,msg,名)` / `PW.assemble()` / `PW.reset()` / `PW.fill(n)` / `PW.count()` / `PW.target` / `PW.version`
- 対応: モバイル / prefers-reduced-motion / IME変換確定 / Esc・focus-visible
